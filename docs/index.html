Architecture

Four small services, one job each

Run them all with one docker compose up, or as separate systemd services. Redis holds live goal state so the gateway can restart without losing a running goal.

gateway-rs
RUST · :9001 · the brain
Where the agents live and the goal pipeline runs. Parses your goal with an LLM, walks it through the state machine, and runs two background loops — a sentinel that checks positions every 5 min and a yield scheduler that hunts better APY every 30 min.
keymaster
RUST · :9100 · the vault
Owns your private keys and is the only thing that ever signs. The gateway never sees a key — it asks keymaster to swap, and keymaster checks the wallet's spend cap, signs, and broadcasts. Keys are encrypted at rest (AES-256-GCM + Argon2id) behind a master password.
nullboiler
ZIG · :9090 · the workflows
A DAG workflow engine with its own SQLite store for multi-step orchestrations — retries, branching, fan-out, sagas with compensation. Fourteen step types, worker dispatch, circuit breakers.
dashboard
PREACT · :9200 · the cockpit
Create goals in plain English, watch every running goal live, browse wallet balances and the tx log, and hit the big red kill switch if everything's on fire.
The swarm

Nine agents, each a focused prompt

They don't run continuously. Each is a small, single-purpose LLM wrapper invoked as the goal moves through its steps — a division of labor, not one giant prompt trying to do everything.

Guardrails

Layers between an LLM and your money

The design goal: nothing can lose you more than you explicitly authorized. In order of defense —

① Wallet caps

Every wallet has a lifetime spend ceiling. Keymaster refuses to sign anything that would push it over the cap.

② Capital clamp

Even if the LLM hallucinates a giant number, the gateway clamps capital to 90% of the wallet cap before the goal starts.

③ Risk gate

Every candidate passes the risk agent first. HIGH risk — low TVL, unverified token, weird pair — means no trade.

④ Stop loss

The sentinel exits any position that drops past your stop-loss percentage, no questions asked.

⑤ Kill switch

One flag halts the swarm. Flip it from the dashboard's big red button or the API.

⑥ Dry-run mode

Go through every step including building the calldata, but never broadcast. Test your prompts without spending gas.

It's still your money, and this is early software. Vespra points an LLM at a live wallet — start on a testnet, run in dry-run first, watch the logs, and never trust it with more than you'd be okay losing.
